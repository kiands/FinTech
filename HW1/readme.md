# 關於雲端運算的思考
## 一些數據
* Amazon在2002年就有意識地佈局雲端運算
* Azure在全球58個國家和地區開展業務，在超過140個國家和地區可以使用
* 阿里雲2009年成立，現在已經建立超過1000PB的具備大數據存儲、分析能力的數據中心
* AWS占有的市場超過雲端運算服務市場的54.1%
* 阿里雲在2020Q3的營收達到107.21億人民幣，份額在2018年達到全球第三，與微軟和亞馬遜相比仍有較大差距（8%）
## 分析
雲端運算是一個燒錢的業務，前期投入，中期維護，善後，吸引客戶都需要操心。
在前期，如何與市場上已有的競爭對手搶占低價的運算硬體非常考驗能力和財力，
中期的電力、冷卻、修繕和汰換則考研運維。在環境保護、廢舊利用和回收上，
也考驗精算師的工作能力。而且光有硬體還不夠。如何將設備組建成集群，
同時又要考慮到靈活性也是一個非常困難的事情。
<br>
雲端運算，往大方向講，有IaaS，PaaS，SaaS。往細節出發，彈性計算，資料庫
存儲，網絡，影片CDN，域名，安全，大數據，人工智慧，物聯網，API等等都是
雲端運算的業務範圍。這看起來似乎有點複雜，開枝散葉，不過我覺得前景可觀
## 展望
現在很多人都想要表現自己。很多公司都想要輕鬆拓展業務。但是由於技術、財力、
人力不足，這些業務剛好可以讓雲端運算服務來承接。而且不需要IaaS，Saas和PaaS
更加符合未來的雲端運算進萬家的潮流。我覺得廠商可以在宣傳上更加注入活力，
推動更多非計算機專業人士運用不同的雲端運算服務展現自己，共建云世代。
<br>
在這個加速發展的世代，我們也需要脚踏實地。雖然業務展望是美好的，但是環境友善
這個話題必須被關注。阿里雲將舊的伺服器轉做輕量級應用伺服器降價發揮餘熱，
Azure嘗試深海冷卻伺服器，都是勇敢的嘗試。縮減浪費、減少不必要的環節，是
我們在發展科技的同時需要考慮的。我們的增量中需要有品質在其中。
